# 새 프로젝트 파일 생성

새로운 포트폴리오 프로젝트를 추가할 때 사용합니다.
`projects/_template.md`를 기반으로 인터뷰 방식으로 정보를 수집한 뒤 파일을 생성합니다.

## 실행 방법

```
/new-project
```

## 커맨드 지침

다음 순서로 실행할 것:

### STEP 1 — 기본 정보 수집
사용자에게 아래 질문을 **한 번에 하나씩** 순서대로 물어볼 것:

1. "프로젝트 이름을 알려주세요. (파일명에 사용될 영문 슬러그도 함께: 예 - content-automation)"
2. "어느 조직/회사에서 진행한 프로젝트인가요? (개인 프로젝트면 '개인'으로 입력)"
3. "프로젝트 진행 기간은? (예: 2024.03 – 2024.08 / 2024.03 – 현재)"
4. "혼자 한 프로젝트인가요, 협업인가요? 협업이라면 본인의 역할은?"
5. "이 프로젝트에서 사용한 기술/도구를 쉼표로 구분해 나열해주세요."

### STEP 2 — 내용 인터뷰
다음 항목들을 **대화형으로** 수집. 각 항목마다 사용자 답변을 확인 후 다음으로 넘어갈 것:

**배경 질문:**
"이 프로젝트 시작 전에 어떤 문제나 비효율이 있었나요? '누군가가 X를 하지 않으면 Y가 발생했다'는 식으로 구체적으로 말해주세요."

**문제 정의 질문:**
"그 문제를 어떻게 재정의했나요? 단순한 불편함 이상으로, 비즈니스 임팩트 관점에서 왜 해결해야 했나요?"

**솔루션 질문:**
"어떤 구조로 해결했나요? 결과보다 설계 방식에 초점을 맞춰서 설명해주세요. 왜 이 방식을 선택했나요?"

**결과 질문:**
"측정 가능한 변화가 있었나요? 수치가 없는 항목은 ◯로 남겨도 됩니다."

**판단 질문:**
"이 프로젝트에서 본인이 직접 내린 중요한 판단이 있었나요? 주어진 요건 실행이 아니라, 본인이 판단하고 결정한 것들이요."

**포트폴리오 메모:**
"이 프로젝트를 어떤 직무에 어필하고 싶으신가요? 혹시 공개하면 안 되는 수치나 정보가 있나요?"

### STEP 3 — 파일 생성
수집된 정보를 바탕으로 `projects/_template.md` 구조에 맞게 파일 생성.

- 저장 경로: `projects/{slug}.md`
- 수치 미확정 항목: `◯` 처리
- 파일 생성 후 전체 내용을 사용자에게 보여주고 수정 여부 확인

### STEP 4 — 우선순위 설정
"이 프로젝트의 포트폴리오 노출 우선순위를 1~5 중 설정해주세요. (1이 가장 중요)"
frontmatter의 `priority` 값 업데이트.

생성 완료 후 출력:
```
✅ 프로젝트 파일 생성 완료
📁 경로: projects/{slug}.md
🏷️ 태그: {tags}
📊 우선순위: {priority}

다음 단계:
- 수치(◯) 확정 후 직접 수정
- /generate-portfolio 로 포트폴리오 생성
```
