# JD 파일 생성

채용 공고를 분석하여 구조화된 JD 파일을 생성합니다.
이후 `/generate-portfolio` 커맨드가 이 파일을 기반으로 채점 기준을 역추산합니다.

## 실행 방법

```
/new-jd
기업명: Channel Talk
공고 내용:
[채용공고 전문 붙여넣기]
```

## 커맨드 지침

### STEP 1 — 입력 파싱
사용자 입력에서 다음을 추출:
- `기업명` (없으면 질문)
- `직무명` (없으면 공고에서 추출)
- `채용공고 원문` 전체

### STEP 2 — 채점 기준 역추산
공고를 분석하여 AI 채점 시스템이 사용할 법한 평가 기준을 역추산할 것.

분석 항목:
1. **필수 기술/경험** (Must-have) — 공고에서 명시적 요구사항 추출
2. **우대 조건** (Nice-to-have) — 우대사항 추출
3. **암묵적 역량** — 직무 설명에서 유추되는 숨겨진 기대치
   - 예: "AI 도입 경험"이 명시되지 않아도 "AI PO" 직무라면 포함
4. **평가 가중치 추정** — 어떤 역량이 더 중요할지 판단 (상/중/하)
5. **포트폴리오 관련 키워드** — 지원자 서류 스크리닝 시 나타날 핵심 키워드

### STEP 3 — 파일 생성
파일명 규칙: `{company-slug}-{role-slug}-{YYYYMM}.md`
예: `channeltalk-ai-po-202504.md`

저장 경로: `jds/{filename}.md`

파일 구조:
```markdown
---
id: {company-slug}-{role-slug}-{YYYYMM}
company: "{기업명}"
role: "{직무명}"
created: "{YYYY.MM.DD}"
deadline: ""
source_url: ""
---

## 채용공고 원문

{원문 전체}

---

## 채점 기준 역추산 (AI 분석)

### 필수 역량 (Must-have)
| 역량 | 근거 문구 | 추정 가중치 |
|------|-----------|-------------|
| ... | ... | 상/중/하 |

### 우대 역량 (Nice-to-have)
| 역량 | 근거 문구 | 추정 가중치 |
|------|-----------|-------------|
| ... | ... | 상/중/하 |

### 암묵적 기대치 (추론)
- ...

### 스크리닝 핵심 키워드
`keyword1` `keyword2` `keyword3` ...

### 한 줄 포지션 요약
이 포지션이 진짜로 원하는 사람: {한 문장 요약}
```

### STEP 4 — 확인
생성된 파일을 사용자에게 보여주고:
"채점 기준 역추산 결과가 적절한가요? 수정이 필요하면 알려주세요."

완료 출력:
```
✅ JD 파일 생성 완료
📁 경로: jds/{filename}.md
🏢 기업: {기업명}
💼 직무: {직무명}

포지션 핵심 키워드: {top 5 keywords}

다음 단계:
- /generate-portfolio jd={filename} 로 포트폴리오 생성
```
