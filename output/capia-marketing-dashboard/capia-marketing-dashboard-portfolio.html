Problem 구조적 문제 3가지
채널 개념 없음 · 링크 목적 혼재 · 사업부 분리 불가 — 3중 구조 문제
01
채널 개념 없는 DB
헤이딜러·엔카 개념이 DB에 없었다
UTM 그룹이 "신차 프로모션"/"중고차 금융" 두 가지로만 분류. 채널(channel_id) FK가 없어 "헤이딜러에서 발생한 신차 전환"을 쿼리할 방법 자체가 없었다.
실제 영향
헤이딜러 클릭 487,951건
어느 버튼에서 왔는지 구분 불가
해결
channel_master 신설 → channel_id FK로 정규화
→ 측정 가능 채널 0개 → 33개
02
링크 목적 혼재
신차 UTM 그룹 안에 중고차 링크가 섞였다
purpose(목적) 속성이 없어 신차 채널이 중고금융 전환을 만들어도 "의도된 전환"인지 "이탈"인지 구분 불가. 크로스전환이 노이즈가 됐다.
실제 영향
편의점·포람페·동호회 등
7개 채널 링크 혼재
해결
purpose ENUM 컬럼 추가 → 신차/중고 탭 분리
→ 크로스전환 자동 측정 가능
03
사업부 분리 불가
신차·중고차 사업부 성과를 따로 볼 수 없었다
신차·중고금융을 한 팀이 담당하는 구조임에도 사업부별 성과를 분리해서 보는 화면이 없었다. 채널 예산 배분 의사결정이 불가능했다.
실제 영향
신차 vs 중고차 채널
ROI 비교 불가 → 예산 조정 불가
해결
B안 섹션 필터 → 신차/중고 탭 분리
→ 사업부별 전환율 독립 측정

Solution 핵심 설계 결정
세 가지 핵심 설계 결정이 전체 구조를 정의했다
결정 01 · DB 정규화
channel_master 신설 — 파싱 대신 정규화
A안 · 미채택
UTM 그룹명에서 채널 파싱 — "네이버GFA", "GFA_커뮤니티애즈" 예외 누적
B안 · 채택
channel_master 테이블 신설 → channel_id FK 참조. 구버전 그룹(259,541건)과 신버전(227,410건) 모두 ch_011로 병합.
→ 채널명 입력 방식에 관계없이 집계 정확

결정 02 · 필터 기준
섹션 탭 — linkType 아닌 purpose(B안)
A안 · 미채택
클릭 전체, 전환만 신차 필터 → 전환율 분모·분자 기준 불일치, 의사결정 불가
B안 · 채택
클릭+전환 모두 purpose="new-car"만 집계. 탭 이름과 데이터 기준 일치. 크로스전환도 측정 가능.
→ 숫자가 작아도 의사결정 가능한 정확성
트레이드오프: 단기적으로 숫자가 더 작게 표시됨
결정 03 · UX 분기
dashboardView — 마케터 설정이 UX를 결정
단일 방식 · 미채택
모든 채널에 링크단위 또는 그룹합산 강제 — 헤이딜러(버튼별 전환율 9.7%~59.1%) 또는 엔카(26개 링크) 둘 중 하나 불편
채택
dashboard_view 컬럼 저장 → 마케터 설정값이 모달 방식 자동 결정. 개발팀 코드 변경 불필요.
→ 어드민 설정 → 대시보드 UX 자동 분기
Data Architecture channel_master DDL 3-Layer 계층
채널 → UTM그룹 → 링크 3단계 계층을 DB에 구축했다

group_type     VARCHAR(20) -- 유료/자연/제휴

요청은 대시보드였지만 — 내가 내린 판단은 모두 구조에 관한 것이었다
01
대시보드 설계보다 데이터 구조 설계를 먼저 했다
요청은 "대시보드를 만들어달라"였지만 현재 DB로는 채널 단위 집계 쿼리가 불가능함을 발견. 기획서보다 DDL과 집계 스키마를 먼저 작성해 개발팀을 설득.
02
마케터 선행 작업을 개발 전에 완료하도록 설계했다
7개 채널 링크 혼재 → 개발팀이 소급 매핑하면 잘못된 데이터 고착 위험. Jira 의존성으로 "마케터 정리 → DB → 소급 → API" 순서 명시.
트레이드오프: 마케터 작업 완료까지 개발 시작 지연
03
헤이딜러 구버전 그룹을 삭제 않고 ch_011로 병합했다
"미사용" 표시 구버전 그룹(클릭 259,541건)이 현재도 유입 중. 삭제 시 과거 데이터 유실 + 현재 트래픽 집계 누락. ch_011로 병합 → 487,951건 전체 집계.
04
섹션 탭 필터를 A안 아닌 B안으로 결정했다
A안(클릭 전체, 전환만 필터)은 숫자가 커 보이지만 전환율이 왜곡. 단기적으로 숫자가 낮아 보여도 의사결정 근거로 쓸 수 있는 정확성 선택.
트레이드오프: 초기 지표가 낮게 표시되어 설득 필요
05
인플루언서 채널을 콘텐츠 그룹에서 설계 시점에 분리했다
지금은 규모가 작아 같은 그룹이어도 무방하지만, 유튜브·인스타 확장 시 플랫폼별 성과 비교 필요. 확장 시점이 아닌 설계 시점에 분리해 나중 리팩토링 방지.

개선 방향
1
API 연동 완료 → 실수치 대시보드 운영
channel_master DDL 기반 집계 API 구현 → MONTHLY_BY_LINK / DAILY_BY_LINK 스키마 연결. 마케팅팀 실시간 조회 시작.
2
채널별 목표 전환율 설정 + OKR 연동
채널별 목표 전환율 입력 → 실적 비교 색상 표시. 마케팅 OKR을 대시보드에서 직접 추적하는 구조.
3
네이버·구글 광고 비용 연동 → CPL 자동 계산
광고 API CPC 데이터 + 전환수 → 채널별 CPL 자동 산출. 예산 배분 의사결정 데이터화.
